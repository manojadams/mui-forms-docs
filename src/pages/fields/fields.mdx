# Fields supported:

## Form fields

The following fields are typically associated with forms and typically contain a value:

-   Text field
-   Number field
-   Email
-   Phone
-   Date
-   Month
-   Password
-   Radio
-   Radio Button
-   Checkbox
-   Select
-   File
-   Search
-   Currency
-   Custom

## Display fields

    The following two fields are used for display purposes only and do not contain any value:

-   Header
-   Label

## Special fields

-   Hidden

## Field properties

    Each field can have the following properties or attributes associated with it:

-   name
-   prop
-   meta
    -   displayName
    -   displayType
    -   displayProps
    -   config
    -   type
    -   options
    -   validation
    -   dependencies
    -   events
-   fields

## General properties

| Field  | Type      | Data type       | Default Value | Description                                                                                           |
| ------ | --------- | --------------- | ------------- | ----------------------------------------------------------------------------------------------------- |
| name   | Mandatory | String          | N/A           | This is the name of the field. It should ideally contain letters and underscore(\_) and no whitespace |
| prop   | Optional  | String or null  | null          | This is the parent property key which is used to group the field while submitting the form            |
| meta   | Mandatory | Object          | N/A           | It contain details about the field                                                                    |
| fields | Optional  | Array of fields | None          | It is used in grouped fields to contain the data belonging to each group                              |
